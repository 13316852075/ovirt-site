# Api

oVirt api package provides Application programming interface for the oVirt engine..

__TOC__

## REST Concept

*   Client–server
*   Stateless
*   Cacheable
*   Uniform interface

## REST Principals

*   Identification of resources
*   Manipulation of resources through representations
*   Self-descriptive
*   Hypermedia as the engine of application state

## oVirt-API URI structure

1. protocol 2. server details 3. entry point (base resource) 4. collection 5. resource 6. sub-collection 7. sub-resource

*   there is one exception from RESTful collection/resource

        architecture – '/capabilities' resource.
